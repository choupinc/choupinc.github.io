Produire une application mobile grace à Microsoft

Qu'est ce que Xamarin.Forms ?

Xamarin est une plateforme open source qui permet developper des applications pour iOS, Android et Windows.
Xamarin gère la communication entre le code et les differentes plateformes visées.
Il permet aux développeurs de produire du code à 90% partagépar les differents systemes d'exploitations mobile grace au C#.

Installation

Pour pouvoir utiliser Xamarin les étapes a suivre sont simples:

  1. Télecharger visual studio 2019 ou 2022
  2. Sur la fenêtre dìnstallation, selctionner "développement mobile en .NET"
  3. Cliquer sur le bouton Installer en bas à droite

Fonctionnement

Xamarin permet de créer une interface utilisateur native sur chaque plateforme et d’écrire une logique dàpplication en C#, qui est partagée entre les plateformes. En général, 80 % du code d’application peut être partagé à l’aide de Xamarin. Le .NET gère automatiquement les tâches concernant que l’allocation de mémoire, le garbage collection et l’interopérabilité avec les plateformes ciblées.

Xamarin.Android

Les applications Android sous Xamarin s’exécutent dans l’environnement d’exécution Mono qui est est écrit avec le langage C. Cet environnement d’exécution s’exécute côte à côte avec la machine virtuelle Android Runtime (ART). Les deux runtimes s’exécutent sur le noyau Linux et exposent diverses API au code utilisateur qui permet aux développeurs d’accéder au système sous-jacent. Il est possible d'utiliser system,System.IO, System.Net et le reste des bibliothèques de classes .NET pour accéder aux installations du système d’exploitation Linux sous-jacentes. Sur Android, la plupart des fonctionnalités système telles que Audio, Graphics, OpenGL et Téléphonie ne sont pas disponibles directement pour les applications natives, elles ne sont exposées que par le biais des API Java Android Runtime résidant dans l’un des espaces de noms Java.* ou les espaces de noms Android.*.

Xamarin.iOS

Les applications iOS sous Xamarin s’exécutent dans le runtime Mono et utilisent une compilation complète avant-temps (AOT) pour compiler du code C# dans le langage d’assembly ARM. Cela est executé côte à côte avec le Objective-C runtime. Les deux runtimes s’exécutent au-dessus d’un noyau de type UNIX, qui est XNU, et relie différentes API au code utilisateur. Cela permettant d’accéder au système natif ou managé sous-jacent.

Qu'est ce que Dotnet Maui ?

Durant mon stage, j'ai eu à faire de la recherche et de l'experimentation concernant Maui et .net 6.Avec l'arrivée du .net 6.
Microsoft a créé une nouvelle plateforme mobile qui remplacera xamamrin dans les années à venir. C'est la version améliorée de Xamarin.Forms.

Fonctionnement

Avec .NET MAUI les API des differentes plateformes sont regroupés sous un seul API permettant de faire appel specifique au composantes native des appareils.
Avec le .NET 6, les differents frameworks tels que Net Framewok, Net core, Net standard, etc n'ont plus de raison d'etre. Le .Net 6 regroupe toutes les librairies: .NET pour Android, .NET pour iOS, .NET pour macOS et Windows bibliothèque UI 3 (WinUI 3). Une bibliothequede classes de base .NET 6 commune a ces frameworks recupère les détails de la plateforme ciblée à contenu dans le code. Pour Android, iOS et macOS, l’environnement est implémenté par Mono, une implémentation du runtime .NET. Sur Windows, Win32 fournit l’environnement d’exécution.

Avantages que Maui ofrre comparé à xamarins

Quelles sont les differences en Xamarin et Maui ?

.Net Maui offre un seul projet pour les differentes platteformes alors qu'avec Xamarin un projets commmun est créé et un projet pour chaque plateforme est rajouté. Cela simplifie la facon de programmer et des gérer des dependance dans un seul projet au lieu de le faire dans differents projets.

Avec .NET MAUI les librairies les plus importantes sont unifiées. Cela permet d'utilider les des fonctionnalités comme celles compris dans xamarin.Essentials (memoire, photo, contact, etc) mais sans avoir a faire appele a ce nuget dans le gestinnaire de paquet.
